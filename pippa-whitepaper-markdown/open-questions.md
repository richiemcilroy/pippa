# Open Questions

These are questions left open by the whitepaper or by the extraction process. No answers have been assumed.

## Resolved

- Product name: Pippa.
- Domain: `pippa.health`.
- Pippa is a mobile app.
- Pippa should launch iOS first, then Android.
- Pippa does not need to be UK-first.
- Girls in Marketing is behind-the-scenes marketing strategy, not a visible in-product launch community by default.
- Manual cycle data entry should be supported.
- Manual cycle onboarding should start with last period start date, average cycle length, period length, and regularity.
- Contraception context and irregular-cycle nuance should be optional settings later.
- Future cycle data integrations should likely start with Apple Health and Health Connect, then map into Pippa's internal data flow.
- Community comes later.
- Share cards should be in the first version for sharing to social media.
- Share cards should hide calories by default, show protein/fibre/progress by default, and allow visibility toggles.
- MVP food logging should include AI/photo logging, barcode scanning, saved meals, text/manual entry, and related whitepaper logging modes.
- Onboarding should require current weight.
- Onboarding should include a fun goal-direction slider while still allowing actual goal weight entry.
- Authentication should be email OTP only for now.
- App Store positioning should use both weight loss and nutrition/consistency language.
- The proposed stack is mostly decided but still open to ideas.
- A selected fact-check pass is wanted.
- Initial selected fact-check pass completed on 2026-05-12.
- Monetisation can be skipped for now.
